| Boundary | Description |
|----------|-------------|
| browser → backend API | All auth requests cross from untrusted client |
| token storage in browser | Where the access/refresh tokens live determines XSS exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | XSS access-token theft | mitigate | Access token in React Context / module-level memory only; refresh token never readable by JS (httpOnly cookie); NO localStorage (D-08) |
| T-03-02 | Spoofing | client-side route guard treated as security | accept-with-control | ProtectedRoute is UX-only; backend require_role is the authority (CM-7); documented in code |
| T-03-03 | Information Disclosure | refresh cookie not sent → infinite redirect | mitigate | axios instance uses withCredentials:true and the SAME instance calls /auth/refresh (Pitfall 5) |
| T-03-04 | Tampering | credentialed requests to wildcard CORS | mitigate | Backend CORS pins http://localhost:5173 (plan 01); frontend relies on it, never assumes wildcard |
| T-03-SC | Tampering | npm installs (vite, shadcn, axios, etc.) | mitigate | All OK-verified in RESEARCH.md Package Legitimacy Audit (sourced from CLAUDE.md); no [ASSUMED]/[SUS]/[SLOP] — no blocking checkpoint required |
