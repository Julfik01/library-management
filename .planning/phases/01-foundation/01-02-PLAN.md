<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /auth/* | Untrusted email/password and cookies cross here |
| client → /admin/* | Role-elevated operations; must be backend-gated |
| JWT → server | Tokens are attacker-influenceable input; algorithm and signature must be validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | JWT algorithm confusion | mitigate | jwt.decode always passes algorithms=["HS256"]; never None (RESEARCH Security Domain) |
| T-02-02 | Information Disclosure | XSS token theft | mitigate | Access token only in response body (client holds in React Context); refresh token in httpOnly cookie; never localStorage (D-08) |
| T-02-03 | Tampering | CSRF on /auth/refresh | mitigate | Refresh cookie set SameSite=Lax, restricting cross-site POST |
| T-02-04 | Elevation of Privilege | refresh token replay after logout | mitigate | Logout inserts token into refresh_token_blocklist; /auth/refresh checks blocklist before issuing (D-04, Pitfall 3) |
| T-02-05 | Elevation of Privilege | student calling librarian/admin endpoints | mitigate | require_role dependency on /admin/users returns 403 (AUTH-07, D-09, CM-7) |
| T-02-06 | Information Disclosure | timing oracle on email existence at login | mitigate | authenticate_user verifies against DUMMY_HASH when user absent |
| T-02-07 | Elevation of Privilege | brute-force login | accept | [ASSUMED] rate limiting out of Phase 1 scope; acceptable for v1 (RESEARCH Security Domain) |
| T-02-SC | Tampering | pip installs | mitigate | No new packages beyond plan 01's audited set; nothing to verify |
</threat_model>
