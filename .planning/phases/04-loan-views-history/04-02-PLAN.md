<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser role state → API request | The page must not trust client-only role hints for access control. |
| search input → backend query | Submitted search text is untrusted and must be bounded and escaped by the backend. |
| page state → list rendering | Pagination inputs must not allow the UI to invent or reorder loan rows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Spoofing | `frontend/src/pages/LoansPage.tsx` | mitigate | Read role from the authenticated context only and rely on backend scoping for authorization. |
| T-04-05 | Tampering / Information Disclosure | `frontend/src/hooks/useLoans.ts` | mitigate | Build requests only from validated page/search state and consume the backend's paginated metadata without client-side slicing. |
| T-04-06 | Denial of Service | `frontend/src/pages/LoansPage.tsx` | mitigate | Keep search submit-only, page size fixed at 10, and reuse response pagination metadata to avoid runaway refetch loops. |
| T-04-SC | Supply Chain | package installs | accept | No new packages are introduced in this phase; reuse the audited Phase 1 frontend stack. |
</threat_model>
