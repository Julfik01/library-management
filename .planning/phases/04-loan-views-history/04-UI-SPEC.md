## Phase Visual & Interaction Contract

- Use one `/loans` page with **Active** and **History** tabs; do not split into separate routes.
- Render both lists as **tables**, not cards or stacked rows.
- **Active** tab sorts by **due soonest first**.
- **History** tab sorts by **newest loan first**.
- Active overdue loans must show a **red overdue badge plus supporting text**; never rely on color alone.
- History rows must show **book title, borrow date, due date, return date, and outcome**.
- Librarian search uses **one query box** that matches **student name or book title**.
- Librarian search runs **on submit only**, not live while typing.
- Librarian results default to **most recent loan first**.
- Librarian result columns prioritize **student name, book title, status, and due date**.
- Pagination uses **numbered pages with next/prev controls**.
- Empty states must be **friendly and explanatory**; never leave a blank table.
