## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars → container | ADMIN_PASSWORD, SECRET_KEY, DB credentials enter the backend container at runtime |
| Alembic migration → DB | Migration writes the seed admin row and all DDL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | SECRET_KEY / ADMIN_PASSWORD in alembic.ini | mitigate | Alembic reads DB URL and secrets from os.environ only; never written to alembic.ini; .env is gitignored, only .env.example is committed |
| T-01-02 | Tampering | available_copies driven negative | mitigate | DB-level CHECK (available_copies >= 0) and (available_copies <= total_copies) created in migration (CP-1) — enforced before any endpoint exists |
| T-01-03 | Elevation of Privilege | Seeded admin with weak/default password | accept | Password sourced from ADMIN_PASSWORD env var (operator-controlled), hashed with argon2; operator responsibility to set a strong value; documented in .env.example |
| T-01-04 | Tampering | invalid role/status values written to DB | mitigate | VARCHAR + CHECK constraints on users.role, borrow_requests.status, loans.status (D-05) reject out-of-set values at DB layer |
| T-01-SC | Tampering | pip installs (requirements.txt) | mitigate | All packages OK-verified in RESEARCH.md Package Legitimacy Audit (sourced from CLAUDE.md authoritative spec); no [ASSUMED]/[SUS]/[SLOP] packages — no blocking checkpoint required |
</threat_model>
